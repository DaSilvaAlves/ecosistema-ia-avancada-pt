P1
Briefing Generator
Transforma a tua dor num PRD base em 2 minutos
briefing-generator.expressia.pt 5 min PRD base
O que fazer
  1. Abre briefing-generator.expressia.pt
  2. Escreve o nome do teu projecto
  3. Escolhe o teu nivel (Iniciante / Intermediario / Avancado)
  4. Descreve a tua dor em 1-2 frases
  5. Lista 3 funcionalidades essenciais
  6. Descreve para quem e a solucao
  7. Clica em "Exportar Briefing JSON"
  8. Cria uma pasta para o teu projecto e dentro dela uma pasta docs/
  9. Guarda o ficheiro exportado como docs/briefing-base.json
Exemplo real — Telmo (Personal Trainer Virtual)
Nome: Personal Trainer Virtual
Nível: Avançado
Dor: "Pessoas terem um personal trainer virtual, porque nem todas conseguem pagar 200/300€ por mês a um personal trainer"
Funcionalidades: planos de treino, acompanhamento, correcção de exercícios/nutrição
Para quem: pessoas que não conseguem pagar 200/300€ por mês a um personal trainer
O que recebes
A ferramenta gera dois blocos:
1. Payload de Orquestração — estrutura técnica do projecto
2. Blueprint Estratégico — directrizes de design e execução

Ambos ficam guardados no teu docs/briefing-base.json. É o teu PRD base.

PÁRA! Não copies estes blocos para o terminal. Ainda não. Falta o passo mais importante: PESQUISA.
Porque nao avancamos ja para o codigo
O Briefing Generator da-te um PRD generico. Nao sabe nada sobre o mercado de fitness virtual, apps concorrentes, precos praticados, modelos de negocio que funcionam. Se construires com isto, vais ter uma app tecnicamente correcta mas comercialmente cega.
P2
Gerar Prompt de Pesquisa
Pede a IA para criar perguntas especificas para o TEU caso
Claude Code / Gemini CLI 5 min Perguntas de pesquisa personalizadas
O que fazer
  1. Abre o terminal (Claude Code ou Gemini CLI)
  2. Cola o prompt abaixo, substituindo os campos a dourado
  3. A IA vai gerar 5-10 perguntas de pesquisa especificas para o teu projecto
  4. Guarda essas perguntas — vais usa-las no Passo 3

Prompt para copiar

Tenho uma ideia de projecto e preciso de fazer pesquisa antes de construir. O meu projecto: [NOME DO PROJECTO] A dor que resolve: [A TUA DOR EM 1-2 FRASES] Publico-alvo: [PARA QUEM E] Funcionalidades principais: [AS 3 FUNCIONALIDADES] Preciso que geres 5-10 perguntas de pesquisa especificas para eu investigar ANTES de construir. As perguntas devem cobrir: 1. MERCADO — Que solucoes ja existem? Quais sao os concorrentes directos e indirectos? 2. PUBLICO — Quem sao estas pessoas exactamente? O que ja tentaram? Porque falharam? 3. MODELO DE NEGOCIO — Como e que projectos semelhantes ganham dinheiro? Precos praticados? 4. FUNCIONALIDADES — Que features sao essenciais vs. nice-to-have? O que diferencia os bons dos maus? 5. TECNOLOGIA — Que APIs, servicos ou integracoes podem acelerar a construcao? Regras: - Perguntas especificas para o MEU caso (nao genericas) - Cada pergunta deve ser pesquisavel numa LLM - Ordena por prioridade (mais importante primeiro)
Exemplo — O que o Telmo receberia
Perguntas geradas pela IA:
1. Que apps de personal trainer virtual ja existem em Portugal e quanto cobram?
2. Qual e o perfil exacto de quem procura PT virtual — idade, rendimento, objectivos?
3. Como e que apps como Freeletics, Nike Training e Fitbod reteem utilizadores?
4. Planos de treino com IA precisam de validacao por profissionais de saude?
5. Que APIs de exercicios/nutricao existem (ExerciseDB, Nutritionix, Spoonacular)?
6. Modelo freemium vs. subscricao: qual converte melhor em fitness?
7. Como e que um PT virtual pode diferenciar-se de conteudo gratuito no YouTube?
P3
Pesquisar em 3 LLMs
Investiga o teu mercado com 3 perspectivas diferentes
Perplexity + GPT + Grok/Gemini 15-20 min 3 conjuntos de resultados
O que fazer
  1. Abre 3 separadores: Perplexity, ChatGPT, Grok ou Gemini
  2. Em cada um, cola as perguntas do Passo 2 (todas de uma vez)
  3. Cada LLM vai dar-te respostas diferentes — isso é o objectivo
  4. Cria uma pasta docs/ no teu projecto (se ainda não existe)
  5. Guarda cada resultado num ficheiro separado:
    docs/pesquisa-perplexity.md
    docs/pesquisa-chatgpt.md
    docs/pesquisa-grok.md
Porque guardar em ficheiros e não copiar para o terminal
Cada pesquisa pode ter milhares de palavras. Se colares 3 pesquisas directamente no terminal, vais estourar o contexto da IA e a qualidade da análise cai. Ficheiros resolvem isto: a IA lê cada um separadamente, com calma, sem perder informação.
Porque 3 e nao 1
Perplexity — pesquisa web real, com fontes. Da-te dados concretos, URLs, numeros.
ChatGPT — analise e insights. Bom a sintetizar e a encontrar padroes.
Grok / Gemini — perspectiva diferente. Pode apanhar angulos que os outros falharam.

3 fontes = triangulacao. Se as 3 dizem a mesma coisa, e verdade. Se divergem, encontraste uma oportunidade.
Nao copies tudo cegamente
Le os resultados. Sublinha o que te surpreendeu. Os insights sao TEUS — a IA pesquisa, tu interpretas. (Principio 8 do ORIGINAI)
P4
Gerar Relatorio Tecnico
Transforma 3 pesquisas num documento unico e estruturado
Claude Code / Gemini CLI 5 min Relatorio tecnico
O que fazer
  1. Confirma que os 3 ficheiros do Passo 3 estão na pasta docs/
  2. Confirma que o docs/briefing-base.json do Passo 1 também está lá
  3. Abre o terminal na pasta do teu projecto (Claude Code ou Gemini CLI)
  4. Cola o prompt abaixo — é curto, a IA vai ler os ficheiros sozinha

A tua pasta neste momento

meu-projecto/ └── docs/ ├── briefing-base.json ← Passo 1 ├── pesquisa-perplexity.md ← Passo 3 ├── pesquisa-chatgpt.md ← Passo 3 └── pesquisa-grok.md ← Passo 3

Prompt para copiar

Lê o meu briefing base em docs/briefing-base.json e as 3 pesquisas de mercado em docs/pesquisa-perplexity.md, docs/pesquisa-chatgpt.md e docs/pesquisa-grok.md Cruza as 3 fontes com o contexto do meu projecto e gera um RELATÓRIO TÉCNICO com estas secções: 1. RESUMO EXECUTIVO (3-5 bullets) 2. ANÁLISE DE MERCADO — concorrentes, preços, gaps identificados 3. PERFIL DO PÚBLICO — quem são, o que querem, o que já tentaram 4. FUNCIONALIDADES VALIDADAS — o que as 3 fontes concordam que é essencial 5. MODELO DE NEGÓCIO RECOMENDADO — baseado no que funciona no mercado 6. RISCOS E OPORTUNIDADES — onde o projecto pode falhar e onde pode brilhar 7. STACK TÉCNICA RECOMENDADA — APIs, serviços, integrações concretas 8. MVP RECOMENDADO — as 3-5 features para a primeira versão Regras: - Cruza as 3 fontes. Se divergem, indica. - Sem opinião — apenas dados e conclusões fundamentadas. - Grava o resultado em docs/relatorio-tecnico.md
O que recebes
A IA grava docs/relatorio-tecnico.md na tua pasta. Um relatório que não existiria se tivesses saltado directamente para o código. Agora sabes:
- Quem são os concorrentes e quanto cobram
- Que features são obrigatórias vs. diferenciadores
- Que modelo de negócio funciona no teu mercado
- Que APIs e serviços podes usar para acelerar

P5
PRD Enriquecido
Combina o PRD base + relatorio tecnico num PRD real
Claude Code / Gemini CLI 5 min PRD final pronto para construcao
O que fazer
  1. No mesmo terminal, cola o prompt abaixo
  2. Inclui o PRD base (do Passo 1) e o relatorio tecnico (do Passo 4)
  3. A IA vai fundir os dois e gerar um PRD informado e fundamentado
  4. Este e o PRD que vais usar para construir — nao o generico do Passo 1

Prompt para copiar

Tenho dois documentos sobre o meu projecto. Preciso que os fundes num PRD final, pronto para construcao. DOCUMENTO 1 — PRD BASE (gerado pelo Briefing Generator): [COLA AQUI O BLUEPRINT DO PASSO 1] DOCUMENTO 2 — RELATORIO TECNICO (baseado em pesquisa real): [COLA AQUI O RELATORIO DO PASSO 4] Gera um PRD FINAL que: 1. Use as funcionalidades VALIDADAS pela pesquisa (nao as genericas) 2. Inclua o modelo de negocio recomendado 3. Use as APIs e servicos concretos identificados 4. Defina o MVP com base no que o mercado pede 5. Inclua criterios de sucesso medidos em numeros reais Formato: - Visao do projecto (2-3 frases) - Problema + publico (com dados da pesquisa) - MVP: funcionalidades com criterios de aceitacao - Stack tecnica com justificacao - Modelo de negocio - Metricas de sucesso Este PRD vai ser entregue directamente ao Claude Code / Gemini CLI para construcao. Deve ser claro, concreto e sem ambiguidade.
A diferenca
PRD generico (Passo 1): "Maximizar a percepcao de valor para o utilizador final. Clean Code e padroes DRY."

PRD enriquecido (Passo 5): "App freemium de treino personalizado com IA. Plano gratuito: 3 treinos/semana gerados por IA com base no perfil. Plano Pro (8EUR/mes): treinos ilimitados + nutricao + progresso. API ExerciseDB para base de exercicios. Diferenciador: planos adaptados ao equipamento que o utilizador TEM (casa vs. ginasio)."
P6
Construir
Entrega o PRD enriquecido a ferramenta de construcao
Claude Code / Gemini CLI / Antigravity 90-120 min App funcional
O que fazer
  1. Cria uma pasta para o teu projecto
  2. Abre o terminal nessa pasta
  3. Cola o PRD enriquecido do Passo 5
  4. A IA comeca a construir — ficheiro a ficheiro, componente a componente
  5. Deixa-a trabalhar. Intervem so quando ela pedir ou quando vires algo errado
  6. Quando tiver uma versao funcional, testa no browser: npm run dev
Que ferramenta usar
Claude Code (CLI) — no terminal, escreve directamente ficheiros. O mais potente.
Gemini CLI — alternativa ao Claude Code, mesmo conceito.
Antigravity (IDE) — interface visual, bom para quem prefere ver o codigo enquanto e gerado.

Todas fazem o mesmo: recebem o PRD e constroem. Escolhe a que te sentires mais confortavel.
Se algo nao funcionar
Nao entres em panico. Copia o erro, cola no terminal e diz: "Este erro apareceu. Corrige." A IA resolve a maioria dos erros sozinha. Se ficar bloqueado, chama o Eurico.
P7
Deploy
Publica a tua app. URL real. O mundo pode ver.
GitHub + Vercel 15-30 min URL publica
O que fazer
  1. No terminal: git init e git add . e git commit -m "primeiro deploy"
  2. Cria um repositorio no GitHub (pode ser privado)
  3. Faz push: git remote add origin [URL] e git push -u origin main
  4. Vai a vercel.com e importa o repositorio
  5. Clica em Deploy
  6. Em 1-2 minutos, tens uma URL publica
Resultado final
Ha 3 horas: "Quero fazer um personal trainer virtual."
Agora: Uma app real, com funcionalidades validadas por pesquisa, modelo de negocio definido, deployed numa URL publica que podes partilhar com qualquer pessoa.

Isto e o metodo ORIGINAI. Clareza antes de execucao. Pesquisa antes de codigo. 90% clareza, 10% delegacao informada.
