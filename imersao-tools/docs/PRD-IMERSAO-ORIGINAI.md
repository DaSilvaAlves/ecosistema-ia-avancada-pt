#### FASE 3 — MAO NA MASSA (15:00-19:00) | 4 horas

Objectivo: cada participante tem o seu projecto real deployed e funcional.

Este e o bloco principal. Cada participante constroi o seu proprio projecto — o problema real que trouxe. Comeca pela dor, gera um briefing, pesquisa o tema com ferramentas de pesquisa, cria um PRD tecnico e entrega a ferramentas profissionais que constroem o projecto.

**Regra:** o participante e o heroi. O Eurico e o guia. As ferramentas trabalham para o participante.

| Hora | Accao | Duracao |
|------|-------|---------|
| 15:00 | Briefing individual — "qual e a tua dor?" (1:1 com Eurico) | 15 min |
| 15:15 | Briefing Generator + pesquisa com ferramentas de pesquisa | 30 min |
| 15:45 | Criar PRD tecnico — "isto e o que queres?" | 15 min |
| 16:00 | Construcao — entregar PRD ao Claude Code, Gemini CLI ou Antigravity | 120 min |
| 18:00 | Primeiro deploy — URL publica | 30 min |
| 18:30 | Iteracao — ajustar, corrigir, melhorar | 30 min |

**Ferramentas:** Claude Code (CLI), Gemini CLI, Antigravity (IDE visual), Perplexity/Grok/GPT (pesquisa), GitHub, Vercel
**Output:** Projecto real deployed e funcional

##### Metodologia ORIGINAI integrada na Fase 3

A Fase 3 segue o metodo ORIGINAI — clareza antes de execucao. O fluxo interno:

```
Expressao (dor do participante)
  → Brief (estruturar a ideia)
    → Detalhamento (etapas e tarefas)
      → PRD tecnico (especificacao para IA)
        → Construcao com ferramentas
          → Deploy
```

O conteudo teorico ORIGINAI (Modulos M0-M6, analogias visuais, templates, prompts) esta disponivel no kit de imersao como material de referencia que o Eurico utiliza durante o guia 1:1.
