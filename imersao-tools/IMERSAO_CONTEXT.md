## 1. O Que é a Imersão IA?
A "Imersão IA" é a academia e metodologia de ensino de **Eurico Alves**.
O objetivo não é construir um software monolítico, mas sim um conjunto de **Micro-Ferramentas (Tools)** que ajudam alunos e criadores a desbloquear o potencial da IA.

**Filosofia:** "Frictionless" (Sem Atrito). Do zero ao protótipo em minutos.

---

## 2. Estrutura do Ecossistema (`imersao-tools`)

### A. 🧠 Student Profiler (`/student-profiler`)
- **O que faz:** Diagnóstico psicológico e técnico do aluno. Define o "Nível de Consciência".
- **Stack:** React 19, Vite, Supabase.
- **Status:** Produção (Online).

### B. 📝 Briefing Generator (`/briefing-generator`)
- **O que faz:** O aluno descreve a sua "Dor" e a ferramenta gera um PRD (Product Requirements Document) técnico e perfeito para entregar a uma IA construtora.
- **Stack:** React, Vite, Tailwind (ou CSS Puro).
- **Status:** Protótipo Funcional.

### C. 🚀 Starter Builder (`/starter-builder`)
- **O que faz:** (Conceito) Um configurador visual que gera o "prompt perfeito" para iniciar projetos em plataformas como Lovable, v0 ou Bolt.
- **Status:** Planeamento.

---

## 3. Regras Técnicas (Tech Standard)
1.  **Stack Preferencial:** Vite + React + TypeScript.
2.  **Estilo:** CSS Modules ou Tailwind (Evitar complexidade desnecessária).
3.  **Backend:** Supabase (BaaS) ou LocalStorage. **NADA DE JAVA/SPRING AQUI.**
4.  **Portas Reservadas:**
    - `5190`: Briefing Generator
    - `5191`: Student Profiler
    - `5192`: Starter Builder
    - *(Evitar as portas 3000-3005 e 8080 do iamenu)*.
